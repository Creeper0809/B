# 项目文件夹结构

这个项目似乎是一个编译器的开发项目，包含多个版本的源代码、构建脚本、测试和文档。

## 根目录

```
/
├── .github/      # GitHub Actions CI/CD 配置
├── bin/          # 存放编译后的二进制文件
├── build/        # 构建过程中生成的临时文件
├── docs/         # 项目文档 (路线图、语法说明等)
├── old/          # 过去版本的源代码和相关文件
├── v3_19/        # 当前正在开发的最新版本 (v3.19)
├── .gitignore    # Git 忽略文件列表
├── GEMINI.md     # Gemini 自定义说明
└── README.md     # 项目主说明文件
```

## 主要目录详解

### `bin/`

存放不同版本编译结果的二进制文件。文件名表明了版本和阶段 (例如 `v3_15_stage0`)。

### `docs/`

包含项目各个方面的详细文档。

- `basm-syntax.md`: `basm` 的语法。
- `v3_syntax.md`, `v4_roadmap.md`: v3 语法和 v4 路线图。
- `vX/`: 各个主版本的特定文档。

### `old/`

包含从 `v1` 到 `v3_18` 的旧版本源代码。每个版本目录通常包含 `src/` (源代码)、`test/` (测试) 和构建脚本。

- `old/v3_15/`:
    - `src/`: v3.15 的源代码。
    - `test/`: v3.15 的测试代码。
    - `build_and_test.sh`: 构建和测试脚本。

### `v3_19/`

当前正在积极开发的版本。结构与 `old/` 中的版本目录类似。

- `src/`: v3.19 源代码。
- `test/`: v3.19 测试。
- `build_and_test.sh`: 构建和测试脚本。
- `config.ini`: 配置文件。
- `syntax.md`: v3.19 语法文档。
